# Smoke Checklist

## Цель
Быстро проверить, что ключевые пользовательские сценарии работают после рефакторинга.

## Подготовка
- Запустить приложение с правами администратора.
- Убедиться, что доступен интернет.
- Закрыть игру перед стартом теста (для сценария "игра не запущена").

## Сценарии

### 1. Старт приложения
- Открыть `PingByDaylight`.
- Ожидание:
  - окно открывается без исключений;
  - список серверов отображается;
  - пинги начинают обновляться.

### 2. Локализация
- Открыть настройки.
- Переключить язык `en` -> `ru` -> `en`.
- Ожидание:
  - основные тексты интерфейса обновляются;
  - приложение не требует перезапуска;
  - настройки языка сохраняются между запусками.

### 3. Gatekeep режим
- Выбрать 2-3 сервера в разных группах.
- Нажать `Apply Selection`.
- Ожидание:
  - изменения применяются без ошибок;
  - в hosts создается/обновляется секция `# -- Ping by Daylight --`;
  - показывается успешное уведомление.

### 4. Reset to default
- Нажать `Reset to default`.
- Ожидание:
  - секция приложения в hosts очищается;
  - появляется успешное уведомление.

### 5. Universal Redirect режим
- В настройках выбрать режим `service`.
- Оставить выбранным ровно 1 сервер.
- Нажать `Apply Selection`.
- Ожидание:
  - hosts содержит перенаправления на выбранный регион;
  - DNS-flush выполняется без падений.

### 6. Проверка обновлений
- Открыть пункт меню проверки обновлений.
- Ожидание:
  - диалог открывается;
  - ошибки сети обрабатываются без падения приложения.

### 7. Connection Info окно
- Открыть окно Connection Info.
- Ожидание:
  - окно открывается и обновляет время/статусы;
  - карточки Lobby/Match отображаются через модульные контролы без визуальных артефактов;
  - кнопки копирования работают, если IP определен.

### 8. Сценарий "игра не запущена"
- Убедиться, что процесс `DeadByDaylight-Win64-Shipping` не запущен.
- Открыть/обновить окно Connection Info.
- Ожидание:
  - статусы показывают "игра не запущена";
  - отсутствуют ошибки в UI.

### 9. Диалоги и action bar
- Открыть `Settings`, `About`, `Check Updates`.
- Ожидание:
  - внизу каждого окна единый action bar (основная/вторичная кнопки);
  - кнопки выполняют прежние действия без регрессий.

### 10. MainWindow список серверов
- Навести мышь на заголовок группы и на элементы серверов.
- Ожидание:
  - hover-подсветка группы и серверов работает;
  - выбор сервера кликом по элементу сохраняется;
  - визуально список не отличается от baseline.

### 11. Командный слой MainWindow
- Использовать меню (`Settings`, `About`, `Check Updates`, `Open Hosts`, `Connection Info`) и нижние кнопки (`Reset to default`, `Apply Selection`).
- Ожидание:
  - действия выполняются через MVVM-команды без regressions в поведении;
  - все окна открываются и закрываются корректно.

### 12. Конфиг `regions.json`
- Временно изменить `Config/regions.json` (например, отключить 1 регион или поменять `groupDisplayName`).
- Ожидание:
  - изменения отражаются в списке регионов после перезапуска;
  - приложение не падает при корректном JSON.

### 13. Ошибка при битом/отсутствующем конфиге
- Временно переименовать `Config/regions.json` или подставить невалидный JSON.
- Ожидание:
  - приложение не стартует в рабочий режим;
  - показывается понятное сообщение об ошибке конфигурации;
  - ошибка зафиксирована в логах.

### 14. Восстановление hosts из шаблона
- Нажать `Reset to default`.
- Ожидание:
  - содержимое hosts берется из `Config/default-hosts.txt`;
  - при отсутствии/пустом `Config/default-hosts.txt` показывается контролируемая ошибка без записи в hosts.

## Чек финализации
- `dotnet restore` успешно.
- `dotnet build` успешно.
- `dotnet test` успешно (если тестовый проект уже есть).
