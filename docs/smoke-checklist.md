# Smoke Checklist

## Цель
Быстро проверить, что ключевые пользовательские сценарии работают после рефакторинга.

## Подготовка
- Запустить приложение с правами администратора.
- Убедиться, что доступен интернет.
- Закрыть игру перед стартом теста (для сценария "игра не запущена").

## Сценарии

### 1. Старт приложения
- Открыть `PingByDaylight`.
- Ожидание:
  - окно открывается без исключений;
  - список серверов отображается;
  - пинги начинают обновляться.

### 2. Локализация
- Открыть настройки.
- Переключить язык `en` -> `ru` -> `en`.
- Ожидание:
  - основные тексты интерфейса обновляются;
  - приложение не требует перезапуска;
  - настройки языка сохраняются между запусками.

### 3. Gatekeep режим
- Выбрать 2-3 сервера в разных группах.
- Нажать `Apply Selection`.
- Ожидание:
  - изменения применяются без ошибок;
  - в hosts создается/обновляется секция `# -- Ping by Daylight --`;
  - показывается успешное уведомление.

### 4. Reset to default
- Нажать `Reset to default`.
- Ожидание:
  - секция приложения в hosts очищается;
  - появляется успешное уведомление.

### 5. Universal Redirect режим
- В настройках выбрать режим `service`.
- Оставить выбранным ровно 1 сервер.
- Нажать `Apply Selection`.
- Ожидание:
  - hosts содержит перенаправления на выбранный регион;
  - DNS-flush выполняется без падений.

### 6. Проверка обновлений
- Открыть пункт меню проверки обновлений.
- Ожидание:
  - диалог открывается;
  - ошибки сети обрабатываются без падения приложения.

### 7. Connection Info окно
- Открыть окно Connection Info.
- Ожидание:
  - окно открывается и обновляет время/статусы;
  - кнопки копирования работают, если IP определен.

### 8. Сценарий "игра не запущена"
- Убедиться, что процесс `DeadByDaylight-Win64-Shipping` не запущен.
- Открыть/обновить окно Connection Info.
- Ожидание:
  - статусы показывают "игра не запущена";
  - отсутствуют ошибки в UI.

## Чек финализации
- `dotnet restore` успешно.
- `dotnet build` успешно.
- `dotnet test` успешно (если тестовый проект уже есть).
